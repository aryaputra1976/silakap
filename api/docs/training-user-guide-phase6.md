# SILAKAP Phase 6 - User Guide

Panduan ini untuk pengguna akhir: Pengelola OPD, Analis, Kabid, dan Kepala Badan.

## 1. Login dan Profil

1. Buka domain SILAKAP.
2. Masukkan username dan password.
3. Jika diminta ganti password, buka `/settings/change-password`.
4. Periksa profil di `/my-profile`.

Tips:

- Jangan membagikan password.
- Gunakan logout setelah selesai memakai komputer bersama.
- Laporkan akun terkunci kepada Admin Sistem.

## 2. Pengelola OPD

### Buat Usulan

1. Buka menu Layanan Kepegawaian.
2. Pilih Buat Usulan.
3. Pilih ASN dan jenis layanan.
4. Lengkapi data pendukung.
5. Upload dokumen wajib.
6. Simpan sebagai draft atau submit.

Checklist sebelum submit:

- ASN benar.
- Jenis layanan benar.
- Semua dokumen wajib terunggah.
- Nomor kontak dan email aktif.

### Memantau Usulan

- Draft Saya: usulan yang belum dikirim.
- Pengajuan Aktif: usulan sedang diproses.
- Dikembalikan: usulan perlu perbaikan.
- Download Hasil: usulan selesai dan memiliki dokumen output.

### Revisi Usulan

1. Buka menu Dikembalikan.
2. Baca alasan pengembalian.
3. Perbaiki data atau dokumen.
4. Submit ulang.

## 3. Analis

### Verifikasi Berkas

1. Buka antrian sesuai role.
2. Buka detail usulan.
3. Periksa ASN, jenis layanan, dokumen, dan riwayat.
4. Pilih approve jika lengkap.
5. Pilih kembalikan/reject jika ada kekurangan, lalu isi catatan jelas.

Catatan verifikasi yang baik:

- Sebutkan dokumen atau data yang perlu diperbaiki.
- Hindari catatan umum seperti "belum lengkap".
- Jika reject, tulis dasar penolakan.

## 4. Kabid

### Approval

1. Buka menu Approval.
2. Periksa hasil verifikasi analis.
3. Approve untuk melanjutkan.
4. Reject atau kembalikan jika keputusan belum dapat diberikan.

### Monitoring

- Beban Kerja: jumlah antrian per tahap.
- SLA Tahapan: konfigurasi dan status warning/overdue.
- Analytics Kabid: SLA trend, throughput, bottleneck, ranking OPD.
- Reject Counter: pemantauan usulan dikembalikan/ditolak.

## 5. Kepala Badan

1. Buka menu TTE & Approval.
2. Periksa usulan yang menunggu TTE.
3. Approve/TTE sesuai kewenangan.
4. Pastikan status usulan menjadi selesai.

## 6. Notifikasi

Notifikasi muncul saat:

- Usulan baru masuk ke tahap berikutnya.
- Usulan dikembalikan/reject.
- SLA mendekati batas atau overdue.
- Dokumen hasil tersedia.

Buka menu Notifikasi untuk membaca dan menandai notifikasi selesai dibaca.

## 7. Bantuan Cepat

Laporkan ke Admin Sistem jika:

- Tidak bisa login.
- Menu tidak sesuai role.
- File tidak bisa diupload.
- Dokumen hasil tidak bisa diunduh.
- Data ASN salah dan perlu peremajaan.
